# AlgoLens — 開発ガイドラインとカスタムコマンド

## 1. 禁止事項・安全対策 (最重要)

- **破壊的変更の禁止**: ファイルの削除、大幅なディレクトリ構造の変更、またはデータベースの初期化などを行う場合は、**必ず事前に実行計画を提示し、ユーザーの許可を得ること**。
- **勝手なコマンド実行の禁止**: `pip install` などの環境操作を伴うコマンド、またはデプロイ関連のコマンドを実行する前に確認すること。
- **推測での実装禁止**: 要件が曖昧な場合は、推測でコードを書かずに、まず質問をして仕様を明確にすること。

## 2. ECCの活用方針

- 新しい機能を追加する際は、必ず `/plan` を使用して事前に設計を共有すること。
- ロジックの実装には `/tdd` (テスト駆動開発) を推奨。
- セキュリティやパフォーマンスに関するベストプラクティスを遵守すること。

## 3. プロジェクトの技術スタック

- 言語: Python 3
- フレームワーク: FastAPI (backend) / Streamlit (frontend)
- テスト: pytest / pytest-cov
- AI: Google Gemini API (`gemini-2.5-flash-lite`)

## カスタムコマンド: 作業終了ルーティン

ユーザーがプロンプトで **「作業終了」** と入力した場合、以下の手順1〜4を自動で実行すること。ユーザーから個別に指示されなくてもよい。

### 手順 1 — 本日の作業内容を自動抽出する

直前の会話履歴を振り返り、`git status` および `git diff HEAD` を実行して変更差分を確認する。
以下の3項目をAI自身で推測・抽出する。

- **今日の実装内容**: 追加・変更した機能や修正点
- **発生したエラーと対策**: エラーメッセージ・原因・解決策のセット
- **技術的なポイント**: 今日の作業で得られた知見・判断の理由

> **補足質問ルール**: AI の推測だけではエラー内容などが十分に分からない場合のみ、追記する前に「今日発生したエラーで特筆すべきことはありますか？」とユーザーに **1つだけ** 質問して情報を補うこと。それ以外の場合は質問せず自動で進めること。

### 手順 2 — `README.md` の開発ログ先頭に自動追記する

`README.md` 内の `## 📖 開発ログ（作業履歴）` セクションを探し、その直下（最初の `<details>` タグの前）に、以下のフォーマットで今日の記録を挿入する。

```markdown
<details>
<summary>📅 YYYY-MM-DD: [AIが考えた今日の作業要約（20字程度）]</summary>

### 🛠 実装内容
- (箇条書き)

### ⚠️ 発生したエラーと対策
- (箇条書き。エラーがない場合は「特になし」と記載)

### 💡 技術的ポイント
- (箇条書き)

</details>
```

- 日付は JST (`Asia/Tokyo`) の当日日付を使用すること。
- 要約タイトルは実装内容を端的に表す日本語（例: `ナレッジベース CRUD 機能の追加`）にすること。
- 既存のログエントリは一切削除・編集しないこと。

### 手順 3 — Git へ自動コミット・プッシュする

追記完了後、以下のコマンドを順番に実行する。

```powershell
git add .
git commit -m "docs: daily update YYYY-MM-DD"
git push origin main
```

コミットメッセージの日付部分は実際の日付（JST）に置き換えること。

### 手順 4 — 完了を報告する

以下のメッセージをユーザーに表示して終了する。

> お疲れ様でした！今日の作業記録をまとめ、GitHub へ Push しました。
