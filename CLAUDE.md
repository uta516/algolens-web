# 開発ガイドラインとガードレール

## 1. 禁止事項・安全対策 (最重要)
- 破壊的変更の禁止: ファイルの削除、大幅なディレクトリ構造の変更、またはデータベースの初期化などを行う場合は、必ず事前に実行計画を提示し、ユーザーの許可を得ること。
- 勝手なコマンド実行の禁止: npm install や pip install などの環境操作を伴うコマンド、またはデプロイ関連のコマンドを実行する前に確認すること。
- 推測での実装禁止: 要件が曖昧な場合は、推測でコードを書かずに、まず質問をして仕様を明確にすること。

## 2. ECCの活用方針
- 新しい機能を追加する際は、必ず /plan を使用して事前に設計を共有すること。
- ロジックの実装には /tdd (テスト駆動開発) を推奨。
- セキュリティやパフォーマンスに関するベストプラクティスを遵守すること。

## 3. プロジェクトの技術スタック
- 言語: Python 3
- テスト: pytest

## 4. 作業終了ルーティン（自動コマンド）
ユーザーがプロンプトで「作業終了」と入力した場合、ユーザーから細かく指示されなくても、以下の1〜4の手順を全自動で実行すること。

1. 直前の会話履歴（解決したエラー等）と `git diff` 等を確認し、「今日の実装内容」「発生したエラーと対策」「技術的なポイント」をAI自身で抽出する。
2. `README.md` の「開発ログ」セクションの先頭に、以下の折りたたみフォーマットで自動追記する。
   <details><summary>📅 YYYY-MM-DD: [AIが考えた今日の作業要約]</summary>

   ### 🛠 実装内容
   - (箇条書き)
   ### ⚠️ 発生したエラーと対策
   - (箇条書き)
   ### 💡 技術的ポイント
   - (箇条書き)
   </details>
3. 追記が完了したら、`git add .` -> `git commit -m "docs: daily update"` -> `git push origin main` を自動で実行する。
4. 最後に「お疲れ様でした！作業記録をまとめ、GitHubへPushしました。」と報告する。
