# 開発ガイドラインとガードレール

## 1. 禁止事項・安全対策 (最重要)
- **破壊的変更の禁止**: ファイルの削除、大幅なディレクトリ構造の変更、またはデータベースの初期化などを行う場合は、**必ず事前に実行計画を提示し、ユーザーの許可を得ること**。
- **勝手なコマンド実行の禁止**: `npm install` や `pip install` などの環境操作を伴うコマンド、またはデプロイ関連のコマンドを実行する前に確認すること。
- **推測での実装禁止**: 要件が曖昧な場合は、推測でコードを書かずに、まず質問をして仕様を明確にすること。

## 2. ECCの活用方針
- 新しい機能を追加する際は、必ず `/plan` を使用して事前に設計を共有すること。
- ロジックの実装には `/tdd` (テスト駆動開発) を推奨。
- セキュリティやパフォーマンスに関するベストプラクティスを遵守すること。

## 3. プロジェクトの技術スタック
- 言語: Python 3 (※必要に応じて変更してください)
- テスト: pytest
